# 在作者信息前添加朝代信息实现计划

## 一、需求分析

1. **篇目列表页面**：为每个篇目条目的作者信息前添加对应的朝代信息
2. **复习模式**：在作者名称前添加相应的朝代信息
3. **显示格式**：统一、美观，不影响原有页面布局和交互功能

## 二、实现步骤

### 1. 篇目列表页面修改

- **文件**：`script.js`
- **函数**：`initTextsList`
- **修改内容**：
  - 在第440行，修改作者信息的显示逻辑
  - 添加朝代信息检查，如存在则显示在作者信息前
  - 格式：`朝代 · 作者`
  - 例如：`唐代 · 李白`

### 2. 复习模式修改

- **文件**：`script.js`
- **函数**：`generateReviewQuestions`
- **修改内容**：
  - 保持现有的联动显示与隐藏逻辑
  - 确保朝代信息显示在作者信息前
  - 格式：`—— 标题 · 朝代 · 作者`
  - 例如：`—— 梦游天姥吟留别 · 唐代 · 李白`

### 3. 空值处理

- 对于没有朝代信息的篇目，只显示作者信息
- 对于没有作者信息的篇目，只显示朝代信息（如果有）
- 确保不显示多余的分隔符

## 三、技术要点

1. **DOM操作**：修改现有的DOM元素内容
2. **条件渲染**：根据朝代和作者信息的存在与否动态生成显示内容
3. **格式统一**：确保两处修改后的显示格式一致
4. **向后兼容**：不影响原有页面布局和交互功能

## 四、预期效果

- 篇目列表页面：每个篇目条目显示为 `标题` + `朝代 · 作者`
- 复习模式：每个背诵卡显示为 `—— 标题 · 朝代 · 作者`
- 两处修改的显示格式统一、美观
- 不影响原有页面布局和交互功能

## 五、文件修改清单

- `script.js`：修改 `initTextsList` 和 `generateReviewQuestions` 函数
